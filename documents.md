# 

以下内容可直接放在知识图谱页面顶部（`knowledge-graph.md` 独立页面中），作为读者的使用指南。

## 侦探笔记：这张图谱不是分类，是关系

> 你面前的这张图，不是一篇文章的目录，也不是一张“谁属于谁”的树状结构。它是一张**思想的活地图**。
> 
> 在“漩涡之在”中，我们反复强调：**关系先于实体**。这张图谱正是这一主张的视觉化实验——重要的不是“文章在哪里”，而是“概念之间如何连接”。
> 
> 你可以拖动它、放大它、搜索它，但不必试图“看懂”全部。真正的意义，不在节点里，而在节点之间的连线上。
> 
> ——短腿兜的零食

---

## 一、这张图谱是什么？

这是一张 **《漩涡之在》的知识关系图谱**。它将书稿中的核心概念、哲学家、案例、文章、章节——以及它们之间的语义关系——编织成一张可交互的网络。

**它不是一张静态的思维导图，而是一个活的、可探索的关系场。**

---

## 二、怎么读？

### 1. 节点：每一种形状和颜色代表不同的东西

| 节点类型     | 颜色  | 形状  | 大小  | 代表什么                                 |
|:-------- |:--- |:--- |:--- |:------------------------------------ |
| **核心概念** | 红色  | 圆形  | 最大  | 漩涡理论的本体论基石（如：关系、漩涡、动态平衡、信息、生成）       |
| **次级概念** | 橙色  | 圆形  | 较大  | 衍生的理论概念（如：上行因果、选择性张力、涌现、自指、僵化）       |
| **哲学家**  | 蓝色  | 圆形  | 中等  | 与漩涡理论对话的思想家（如：德勒兹、贝特森、怀特海、齐泽克）       |
| **案例**   | 绿色  | 三角形 | 中等  | 书稿中分析的文学/影视/社会案例（如：《四个春天》、赤名莉香、开源社区） |
| **文章**   | 紫色  | 圆形  | 最大  | 具体的博客文章（简版书稿、对话篇、人文篇中的每一篇）           |
| **章节**   | 绿色  | 圆形  | 较小  | 文章内部的 h1/h2/h3 标题，点击可跳转到该节           |
| **分类**   | 橙色  | 圆形  | 较小  | 文章所属的分类（如：哲学对话、人文篇、简版书稿）             |
| **标签**   | 蓝色  | 圆形  | 较小  | 文章的技术标签（如：哲学、哲学对话、漩涡之在）              |

**大小提示**：一个节点连接越多（关系越密集），它在图谱中显示得越大。

### 2. 连线：节点之间的语义关系

连接线带有一个标签，说明两个节点之间的关系类型：

| 关系类型   | 含义              | 示例              |
|:------ |:--------------- |:--------------- |
| **构成** | 概念之间的本体论构成关系    | 信息 → 关系（构成）     |
| **奠基** | 哲学家为某个概念奠定了基础   | 贝特森 → 信息（奠基）    |
| **共鸣** | 哲学家的思想与漩涡理论高度一致 | 德勒兹 → 生成（共鸣）    |
| **批判** | 哲学家的思想与漩涡理论形成对照 | 齐泽克 → 自指（批判）    |
| **印证** | 案例验证了某个概念的有效性   | 赤名莉香 → 动态平衡（印证） |
| **缠绕** | 两个概念不可分割        | 信息 ↔ 生成（缠绕）     |
| **循环** | 双向因果关系          | 上行因果 ↔ 下行因果（循环） |
| **递进** | 概念的层级跃迁         | 自组织 → 自创生（递进）   |
| **张力** | 两个概念之间的内在紧张关系   | 肯定性 ↔ 否定性（张力）   |
| **超越** | 理论对旧范式的超越       | 实体论 → 关系第一性（超越） |
| **对立** | 对立关系            | 僵化 ↔ 韧性（对立）     |

### 3. 图例

图谱左下角（或底部）的图例可以隐藏/展开，帮助你快速定位不同颜色的节点。

#### 4.图谱几种显示情况：

| 情况                             | 是否显示为节点   | 是否有边连接             |
|:------------------------------ |:--------- |:------------------ |
| 在概念定义中，且在文章概念中出现 如concept[有 ]    | ✅ 是       | ✅ 有（文章→概念 + 概念→概念） |
| 在概念定义中，但无文章引用concept[ 无 ]，也无概念关系 | ✅ 是（孤立节点） | ❌ 无                |
| 在概念定义中，但无文章引用concept[ 无 ]，却有概念关系 | ✅ 是       | ✅ 有（概念→概念）         |
| 不在概念定义中，却在文章概念中出现concept[有 ]     | ❌ 否（不会显示） | ❌ 无                |

>注：文章和章节节点总是会显示，和文章头部concept[]概念无关。
>
> 注：定义指的是图谱代码中CONCEPT_DEFINITIONS。只有CONCEPT_RELATIONS定义不会显示节点。

## 三、怎么操作？

### 1. 搜索

在顶部的搜索框中输入**文章标题、概念名、哲学家名、分类或标签**，图谱会高亮显示所有匹配的节点及其直接邻居，其他节点暂时淡出。

**提示**：搜索“德勒兹”，你会看到所有与德勒兹相关的概念和文章同时高亮，形成一张“德勒兹在漩涡中的关系网”。

### 2. 点击跳转

- **紫色文章节点**：点击跳转到对应的博客文章。
- **绿色章节节点**：点击跳转到文章中的 h1/h2/h3 标题位置。

### 3. 拖拽与缩放

- **拖拽节点**：可以手动调整节点的位置，方便查看局部关系。
- **滚轮缩放**：放大查看密集区域，缩小查看全局结构。
- **空白区域拖拽**：移动画布视角。

## 四、这张图谱的“底层逻辑”

这张图谱的构建，本身就是在演示“漩涡之在”的核心理念：

1. **关系第一性**：重要的不是“文章”或“概念”本身，而是它们之间的连接方式。你在图谱中看到的最有信息量的东西，是那些带标签的连线。
2. **信息是制造差异的差异**：节点大小、颜色、形状的差异化视觉编码，本身就是一种信息——它告诉你“哪个概念更重要”“哪个节点连接更密集”。
3. **连接是差异的事件化**：当你点击一个节点，它高亮并显示所有邻居，这个过程就是一次“关系的激活”。图谱不是静态的，它在你的每一次交互中重新生成。

## 五、维护与更新

- 目前共有 **109 个概念节点**（核心10个 + 次级48个 + 哲学家30个 + 案例17个 + 东方4个），**76 条预定义语义关系**，以及由文章动态生成的文章、分类、标签、章节节点。
- 新增文章时，在文章头添加 `concepts` 字段即可让文章自动连接到图谱中已有的概念节点。
